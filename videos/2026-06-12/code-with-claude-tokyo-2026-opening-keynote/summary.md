## 一句話總結
Anthropic 在東京 Code with Claude 大會上發表第五代模型 Claude Fable 5 與 Mythos 5，並圍繞「模型、平台、產品」三層展示如何用 Claude Managed Agents 與 Claude Code，把指數成長的 AI 能力轉化為真正的商業成果。

## 重點摘要
- 開場由雲端平台工程主管 Caitlyn 揭示 Anthropic 是「平台公司」，平台 API 年用量成長近 17 倍。
- 剛發表的 Fable 5（最強公開模型）與 Mythos 5 同源；能力跳躍越來越大、間隔越來越短，但企業落地仍呈線性，形成待補缺口。
- Fable 5 強在「單發即正確」與「長時程自主」（跨數天、數百萬 token、可指揮子代理），讀程式／追故障的能力比寫程式更突出。
- 安全機制把資安／生化請求路由到 Opus 4.8 並標示計價，讓最強能力今天就能交付；Mythos 5 為解除防護版、僅限 Glass Wing 夥伴。
- Angela 提出 AI-native 三要素（harness／context／infrastructure），整合為 Claude Managed Agents，今日上線排程部署與 vault 環境變數。
- Cat 展示 Claude Code 從 CLI 到 Cloud Desktop 的多介面、multi-claude 並行，以及 code review、routines、Claude Security、dynamic workflows。
- 客戶實證：Rakuten、Canva、Spotify（遷移時間砍 90%）、Mercari（產出年增 90%）。

## 關鍵結論／takeaways
- 為「下一版 Claude」設計架構與 eval，當卡住的任務開始成功就是出貨訊號。
- 模型越強越該給簡單原語＋強模型，少用過度複雜的 harness。
- 把模型升級當商業機會，建立自動化 evals 讓升級容易。
- 想搭長時程代理優先用 Managed Agents；dynamic workflows 與 Fable 5 今日皆可上手。
